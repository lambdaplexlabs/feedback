# Lambdaplex Feedback

This repository is the **public issue tracker** for [Lambdaplex](https://lambdaplex-labs.gitbook.io/lambdaplex) — a Hedera-native onchain exchange protocol built by Lambdaplex Labs.

It exists **only** to collect bug reports and feature requests. The Lambdaplex source code is not hosted here.

## 📌 What this repo is for

- 🐞 **Reporting bugs** in the Lambdaplex protocol, matching service, SDK/API, or web app
- 💡 **Requesting features** or improvements

## 🧭 Before you open an issue

1. **Read the [docs](https://lambdaplex-labs.gitbook.io/lambdaplex)** — your question may already be answered there.
2. **[Search existing issues](https://github.com/lambdaplexlabs/feedback/issues?q=is%3Aissue)** — if your bug or idea already exists, give it a 👍 and add details in a comment instead of opening a duplicate.
3. **Pick the right channel** (see below).

## 🚦 Where things go

| You want to...                          | Use this                                                                 |
| --------------------------------------- | ------------------------------------------------------------------------ |
| Report a reproducible bug               | [New issue → Bug Report](https://github.com/lambdaplexlabs/feedback/issues/new/choose) |
| Request a feature                       | [New issue → Feature Request](https://github.com/lambdaplexlabs/feedback/issues/new/choose) |
| Ask a question / get help               | [Discussions](https://github.com/lambdaplexlabs/feedback/discussions)    |
| Report a **security vulnerability**     | **Do not open an issue** — see [SECURITY.md](./SECURITY.md)              |

> ⚠️ **Security note:** Lambdaplex is a DeFi protocol. If you've found a vulnerability that could affect funds, contract integrity, or user safety, **report it privately** via our [security policy](./SECURITY.md). Public disclosure puts users at risk.

## 📥 How issues are handled

- New issues are labeled `needs-triage` and reviewed by the Lambdaplex Labs team on a regular cadence.
- We may move questions or open-ended discussion into [Discussions](https://github.com/lambdaplexlabs/feedback/discussions) to keep the tracker focused.
- This repository does **not** accept pull requests — there's no code here to change.
